## 为什么需要自建 CDN

中国境内访问 GitHub / Microsoft Store / Ubuntu 镜像速度极慢甚至不可达：

| 境外源 | 国内实测 | CDN 替代 |
|--------|----------|----------|
| `github.com/NousResearch` | 不可达 | `hermes-agent.tar.gz` |
| `raw.githubusercontent.com` | 不可达 | `hermes-install-standalone.sh` |
| `deb.nodesource.com` | 极慢 | `setup-node22.x` |
| `wslstorestorage.blob.core.windows.net` | 偶尔可达 | `wsl_update_x64.msi` |
| `github.com/microsoft/WSL/releases` | 不可达 | `wsl.2.6.3.0.x64.msi` |
| `cloud-images.ubuntu.com` | 偶尔可达 | 直接用新 URL `cdimages.ubuntu.com` |

## 如何更新 CDN 文件

```bash
# 1. 放入文件
cp your-file /www/wwwroot/hermes-cdn/files/

# 2. 更新校验
cd /www/wwwroot/hermes-cdn/files
sha256sum * > files.sha256

# 3. 确认
curl -I http://121.40.165.216/hermes-cdn/files/files.sha256
# HTTP/1.1 200 OK
```

## 脚本如何使用 CDN

`install-hermes.ps1` 中的配置：

```powershell
$Script:CDN_BASE = "https://121.40.165.216/hermes-cdn/files"
$Script:MIRRORS = @{
    apt = "https://mirrors.tuna.tsinghua.edu.cn/ubuntu/"
    pip = "https://pypi.tuna.tsinghua.edu.cn/simple"
    npm = "https://registry.npmmirror.com"
}
```

所有下载操作优先走 CDN，失败时回退到原始 URL。
