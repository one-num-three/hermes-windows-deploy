## 三、维度 2：逻辑完整性（✅ 本机可测）

### 3.1 脚本参数传递链路

测试 `install-hermes.ps1` 的参数是如何一路传到各 shell 脚本的。

```
用户参数
  ├── -Port "8648"      → $Script:WEB_PORT → $WSL_USER 的 bash heredoc → $PORT
  ├── -WslPath "D:\WSL"  → Step-InstallUbuntu 的 $WslPath
  ├── -UbuntuUser "xxx" → $Script:WSL_USER → useradd 命令
  └── -SkipWsl          → Main 的 step 列表过滤
```

| 测试点 | 方法 | 预期 |
|--------|------|------|
| Port 参数传递 | 静态审查 heredoc 中 `$PORT` 变量来源 | 从参数一路到 systemd 服务 |
| 密码生成 | 脚本中包含密码生成逻辑 | 12 位，含大小写+数字 |
| 断点续装状态 | 审查 `Set-InstallState` / `Test-StepDone` 逻辑 | step 失败后状态="failed"，不会误判"done" |

### 3.2 安装步骤顺序依赖

## 四、维度 3：集成连通性（✅ 本机可测）

### 4.1 CDN 可达性

```bash
BASE="http://121.40.165.216/hermes-cdn/files"

# 测试每个文件的 HTTP 状态
for f in hermes-agent.tar.gz hermes-install-standalone.sh setup-node22.x wsl_update_x64.msi; do
    code=$(curl -sI -o /dev/null -w "%{http_code}" "$BASE/$f" --max-time 5)
    size=$(curl -sI "$BASE/$f" --max-time 5 | grep -i content-length | awk '{print $2}' | tr -d '\r')
    if [ "$code" = "200" ]; then
        echo "✓ $f — HTTP $code — ${size} bytes"
    else
        echo "✗ $f — HTTP $code"
    fi
done
```

## 八、本机可执行的快速测试脚本

```bash
#!/bin/bash
# 一键自检脚本 — save as /tmp/hermes-self-test.sh
# 在服务器上执行: bash /tmp/hermes-self-test.sh

PROJECT="/www/wwwroot/hermes-windows-deploy"
CDN_BASE="http://121.40.165.216/hermes-cdn/files"
PASS=0
FAIL=0

check() {
    if eval "$1" &>/dev/null; then
        echo "  ✓ $2"
        ((PASS++))
    else
        echo "  ✗ $2"
        ((FAIL++))
    fi
}

echo "=== Hermes 项目自检 ==="
echo ""

echo "--- 语法检查 ---"
for f in "$PROJECT/scripts/"*.sh; do
    check "bash -n '$f'" "$(basename $f)"
done
for f in "$PROJECT/desktop/chat/server/"*.js "$PROJECT/desktop/hitl/server/"*.js; do
    check "node -c '$f'" "$(basename $f)"
done

echo ""
echo "--- CDN 连通性 ---"
for f in hermes-agent.tar.gz hermes-install-standalone.sh setup-node22.x wsl_update_x64.msi; do
    code=$(curl -sI -o /dev/null -w "%{http_code}" "$CDN_BASE/$f" --max-time 5 2>/dev/null)
    if [ "$code" = "200" ]; then
        echo "  ✓ $f (HTTP 200)"
        ((PASS++))
    else
        echo "  ✗ $f (HTTP $code)"
        ((FAIL++))
    fi
done

echo ""
echo "--- nginx 配置 ---"
check "nginx -t" "nginx 配置语法"

echo ""
echo "--- 关键文件完整性 ---"
for f in \
    "scripts/install-hermes.ps1" \
    "scripts/wsl-bootstrap.sh" \
    "scripts/setup-mirrors.sh" \
    "scripts/setup-systemd.sh" \
    "scripts/integrate-desktop.sh" \
    "scripts/post-install.ps1" \
    "scripts/uninstall.ps1" \
    "scripts/build.ps1" \
    "desktop/chat/server/chat-endpoint.js" \
    "desktop/chat/client/ChatPage.vue" \
    "desktop/hitl/server/approval-ws.js" \
    "desktop/hitl/client/ApprovalPanel.vue" \
    "desktop/mcp/ToolHub.vue" \
    "desktop/shell/register-shell.ps1" \
    "desktop/shell/send-to-hermes.ps1" \
    "gui/HermesInstaller/HermesInstaller.csproj" \
    "installer/hermes-installer.iss" \
    "实现计划书.md" \
    "README.md"; do
    check "[ -s '$PROJECT/$f' ]" "$f"
done

echo ""
echo "================================"
echo "通过: $PASS  失败: $FAIL"
echo "================================"
```
