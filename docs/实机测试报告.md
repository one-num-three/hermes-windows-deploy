## 测试流程与每步结果

### Step 0: 环境检测 — ✅ 通过

**成功:**
- 磁盘检测: 25.9GB > 5GB ✅
- 内存: 1.9GB → 警告（建议 8GB），但不阻断
- WSL 命令可用 ✅

**发现 Bug #1**: `Get-CimInstance Win32_Processor` 在 1vCPU 云实例上返回 null，导致 `.Name.Trim()` 抛出 `InvokeMethodOnNull`。
**修复**: 加 `-ErrorAction SilentlyContinue` + null 检查，失败时优雅降级。

**发现 Bug #2**: UTF-8 无 BOM 的 PowerShell 脚本通过 scp 传输到中文 Windows 后，被 PowerShell 按 GBK 解析，所有中文注释变为乱码，破坏语法结构。
**修复方案**: 传输前给 `.ps1` 文件加 UTF-8 BOM (`\xef\xbb\xbf`)，或使用 stdin 管道 + `[Console]::InputEncoding`。

### Step 1: WSL2 安装 — ✅ 通过（但需要更新）

**成功:**
- wsl.exe 已存在 ✅
- `.wslconfig` 配置完成 ✅

**发现 Bug #8**: 当前 WSL 是 Windows Server 2022 内置旧版（10.0.20348.2849），不支持 `--import` 和 `--version` 等现代化标志。执行 `wsl --import` 时显示帮助信息并返回 exit 0（误判为成功）。
**修复**: 新增 WSL 版本检测逻辑——解析 `wsl --help` 输出，若不包含 `--import` 则自动从 CDN 下载并安装 WSL 2.6.3 MSI，然后重启。

### Step 2: Ubuntu 安装 — 两次尝试

**尝试 A（默认 Store 路径）**: ❌
- Windows Server 2022 无 Microsoft Store
- `wsl --install -d Ubuntu-24.04` 失败
- 错误: "WSL Ubuntu 安装失败，请手动安装"

**尝试 B（-WslPath 自定义路径）**: ⚠️（下载成功，导入失败）
- **Bug #7**: rootfs URL `https://cloud-images.ubuntu.com/wsl/noble/current/ubuntu-noble-wsl-amd64-rootfs.tar.gz` 返回 404
- **修复**: 更新为 `https://cdimages.ubuntu.com/ubuntu-wsl/noble/daily-live/current/noble-wsl-amd64.wsl`
- 376MB 下载成功 ✅
- `wsl --import` 导入失败 ❌ — 旧版 WSL 不支持 `--import`（见 Bug #8）

### Step 3-5: 引导/Hermes/WebUI — 因 Ubuntu 未安装跳过

### Step 4 脚本解析 — 发现 3 个 Blocker Bug

**Bug #4**: `\$` 在 PowerShell 双引号中不是转义符。脚本中 3 处双引号 here-string 使用 `\$VAR` 试图阻止变量展开，但 `\` 在 PS 中不是转义字符，`$` 仍触发展开，导致 `$WSL_USER:` 被解析为无效的驱动器变量引用，抛出 `InvalidVariableReferenceWithDrive`。

**修复**: 将双引号 here-string `@"..."@` 改为单引号 `@'...'@`（不展开任何变量），通过 `.Replace('__PLACEHOLDER__', $value)` 注入 PowerShell 变量。

**Bug #5**: `-f` 格式化操作符与 bash awk 命令中的 `{print $NF}` 花括号冲突。
**修复**: 弃用 `-f`，全部改用 `.Replace()` 方法。

**Bug #6**: `.Any({...})` 是 LINQ 扩展方法，PowerShell 不支持将 ScriptBlock 隐式转换为 `Func<T,bool>`。
**修复**: 改用 `[string].IndexOfAny(char[])` 纯 .NET 方法。

## 环境限制说明

此测试机 **1.9 GB 内存** 低于 Hermes 生产推荐配置（8 GB），导致：
- WSL2 内存限制设为 0.5 GB（最低配置）
- 完整 Ubuntu + Hermes 安装未能在本次测试中完成（WSL 需先更新并重启）

脚本在 ≥ 4 GB 的机器上预期可完整安装。

## 补充发现（R8.5 轮次）

### Bug 9 — WSL 版本检测 `--help` 输出 UTF-16LE 乱码
`wsl --help` 输出经 PowerShell `Out-String` 后中文变乱码，导致 `-match '--import'` 始终为 $false。
**修复**: 改用 `(Get-Item wsl.exe).VersionInfo.FileVersion` 版本号比较（≥ 2.0）。

### Bug 10 — 新版 WSL 安装后 PATH 优先级问题  
WSL MSI 安装到 `C:\Program Files\WSL\wsl.exe` 但 System32 中的旧版仍被优先调用。
**修复**: 脚本启动时检测 `C:\Program Files\WSL\wsl.exe` 并 prepend 到 PATH。

### CDN 协议问题
`$Script:CDN_BASE = "https://..."` — CDN 无 SSL 证书，HTTPS 请求在 PowerShell 层失败。
**修复**: 改为 `http://`。

### 新增 CDN 文件
- `ubuntu-noble-wsl-amd64.wsl` (376MB) — Ubuntu 24.04 WSL 镜像（下载中），用于加速国内用户下载

### 环境限制确认
- 阿里云 ECS 不支持嵌套虚拟化 → WSL2 服务无法启动
- `cdimages.ubuntu.com` 从国内下载极慢（~17KB/s）
- 建议：使用支持嵌套虚拟化的实例 + Ubuntu rootfs 走自建 CDN
